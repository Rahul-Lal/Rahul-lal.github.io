GitHub Fork & Clone

What are some examples of when you would fork?
An example I can think of is when you want to create you own version of a project, and don't want the source code of your project be commited on the repository of the project it's based on

Write your own step-by-step fork and clone instructions. Describe what you did.
Forking:

  1. Login into GitHub
  2. Find desired repository through the author's profile or use search engine
  3. Once found, press the Fork button on the top right-hand corner of the window
  4. Select the location where you want the forked repository located


Cloning:

  1. Open the desired repository
  2. On the middle right hand corner, press the green button 'Clone or download'
  3. Press the button next to the repository's link
  4. Close the window & open Git Bash on your computer
  5. Direct yourself the intended directory
  6. Enter 'git clone Enter repository link.git', then press Enter

Thinking like a programmer

Describe the process for solving problems
According to a blog on the site Zapier, their problem solving process for programmers follow these two steps:

  1. Find the answer independently
  2. Ask a community a more seasoned individuals


Another blog from Codeburst has 10 steps:

  1. Read the problem at least three times
  2. Manually work through the problem with at least three sets of sample data
  3. Simplify & optimize the solution
  4. Write solution in pseudocode (dummy code)
  5. Convert pseudocode into actual code of choice and debug it
  6. Simplify & optimize the code of the solution
  7. Debug the issues with solution
  8. Write comments, really useful ones
  9. Get feedback by reviewing code from someone else
  10. Practice makes perfect

Will you follow that process? How?
I will more likely follow the ten step process from Codeburst, as it more robust process which allows me to solve the problem almost independently.
