As I'm talking to a Non-technical friend, I'll discuss:

An analogy to describe the differences between HTML and CSS
Think of a website as a person, a human being:

HTML: The physical being and the parts such as hands, feet, chest, head, hair (optional), eyes, nose, mouth, etc
CSS: How the being is presented such as eye color, hair color & style, size of hands & feet, skin color, their height, weight. So basically what they look like

Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'. (But not those ones).
Control Flow is the order how functions, variables & object declarations, and them being called are executed and evaluated in a running programming. Think of the order you follow a recipe on a cookbook
Loops are more a repetitive, where a function does a task/set of tasks for more than once, whether they know how many times or not. An example I can think of is when a school teacher grades a stack of their student's test, one by one.

Describe what the DOM is and an example of how you might interact with it.
Document Object Model or DOM is a programming API is designed to understand a HTML & XML file. Understand the structure of the elements and how it's accessed or manipulated.
An example I can think of is that features on Microsoft Excel how they convert the data on a spreadsheet into a visual graph like a pie or bar graph.

Explain what functions are and why they are useful.
A function is basically a series of tasks, like a recipe in a cook. The value it beings, is that it does a number of task within one tasks, killing as many birds with a singular stone.

Explain the difference between accessing data from arrays and objects.
Accessing data from a array is like choosing from a range of options, like the options at a fast food outlet.
Accessing data from a object is like picking a aspect of the object, think remembering someone from the shirt they wear, or their name, or any small detail of a person they you found memorable.
